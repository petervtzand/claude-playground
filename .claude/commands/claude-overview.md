Generate (or overwrite) `CLAUDE_OVERVIEW.md` at the repo root. It is a quick reference of every Claude Code primitive available in this project: what each thing is in one sentence, and what's actually set up here.

Keep it terse. No verbose explanations, no markdown links, no inline file paths beyond what a bullet point needs.

## Required structure

Top of file: one-line intro saying this is auto-generated and is regenerated by `/claude-overview`.

Then one H2 section per primitive, in this order:

1. CLAUDE.md scopes (project / local / user-global)
2. Skills
3. Slash commands
4. Subagents
5. MCP servers
6. Hooks
7. Settings (`settings.json`)
8. Workflows (`/loop`, `/schedule`)
9. Plugins (Superpowers)
10. Tooling around the repo (Taskfile, ruff, pyright, uv) — *not* a Claude primitive but listed for completeness

## Format for each section

Use H3 subheaders, not bold:

```
### What & when
One sentence. Tutor tone.

### Currently set up
- artifact-name (short when-to-use parenthetical)
- another-artifact (short when-to-use parenthetical)
```

If nothing is configured, write a single bullet `- None` (or `- Not installed` / `- No custom X yet`, whichever fits).

### Special case: CLAUDE.md scopes

For each CLAUDE.md file that exists, list it as a bullet with the same parenthetical, and nest its top-level section headings as sub-bullets (just the heading text, no descriptions):

```
- CLAUDE.md (team-shared codebase facts, committed)
  - Purpose
  - Commands
  - Architecture
  - Working mode
```

## How to gather state

Read these before writing:

- `CLAUDE.md`, `CLAUDE.local.md` (if present), `~/.claude/CLAUDE.md` — for the section list under "CLAUDE.md scopes"
- `.claude/commands/`, `.claude/skills/`, `.claude/settings.json`, `.claude/settings.local.json`, `.mcp.json` (any that exist)
- `~/.claude/skills/` — for personal skills
- `pyproject.toml`, `Taskfile.yml` — for the tooling section
